# Project: World Cup slot frontend (Stake Engine)

This repo is the StakeEngine web-sdk (SvelteKit + PixiJS + xstate, TurboRepo,
pnpm 10.5.0, Node >= 22.16.0 — README's "18.18.0" is stale; stay inside WSL).

## What we're building

The frontend for **World Cup**, a finished math game living in the sibling repo
`~/projects/math-sdk/games/0_0_worldcup` (readme.txt there has the full spec).
Plan: copy `apps/lines` (matching mechanic) to a new app and re-theme it.
The user is a learner practicing integration — work step-by-step WITH them,
explain decisions, let them drive design choices. Do not bulk-complete steps.

## Game spec the frontend must honor

- 5 reels x 4 rows, 20 paylines, lines-pay, RTP 0.97, wincap 5000x,
  very high volatility (bonus-driven; make scatter anticipation + bonus
  celebrations feel big).
- Bet modes: `base` (cost 1.0), `bonus` buy (cost 100.0, forces freespins).
- Freespins: 3/4/5 scatters -> 8/12/15 spins; retrigger 2+ in freegame.
  Freegame wilds carry multipliers (2x-50x) that add and multiply line wins.
- Book events used by the math: reveal, winInfo, setWin, setTotalWin,
  freeSpinTrigger, updateFreeSpin, finalWin (see books in
  `~/projects/math-sdk/games/0_0_worldcup/library/books/`).
- Frontend config generated by math:
  `~/projects/math-sdk/games/0_0_worldcup/library/configs/config_fe_0_0_worldcup.json`

## Symbol art mapping (11 sprites needed)

W=Wild, H1=Trophy, H2=Golden Boot, H3=Star Jersey, H4=Ref Cards,
L1=Yellow Ball, L2=Red Ball, L3=Blue Ball, L4=Orange Ball, L5=Green Ball,
S=Scatter (art still TBD from artist).
